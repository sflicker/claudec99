# ClaudeC99 Stage 108 — `#elifdef` / `#elifndef` preprocessor directives

## Goal

Add two new conditional-compilation directives that are widely used in
system headers and real-world C code:

```c
#elifdef  NAME    /* equivalent to: #elif defined(NAME)  */
#elifndef NAME    /* equivalent to: #elif !defined(NAME) */
```

Both directives are branch-transition forms that sit between an existing
conditional block (`#ifdef` / `#ifndef` / `#if`) and the final `#else` or
`#endif`.  They were standardised in C23 (N2645) and have been supported by
GCC since version 12 and Clang since version 13; many current system headers
already emit them.

The change is entirely confined to **`src/preprocessor.c`** — no lexer,
parser, AST, codegen, or header changes are needed.  Version is bumped to
`01080000`.

---

## Background — current gap

The preprocessor's directive dispatch in `preprocess_internal`
(`src/preprocessor.c`) handles `#ifdef`, `#ifndef`, `#if`, `#elif`,
`#else`, and `#endif`.  A `#elifdef` or `#elifndef` line is not recognised;
it falls through to the unsupported-directive error:

```
error: unsupported preprocessor directive
```

Real-world headers that use `#elifdef` (e.g. headers generated by
`autoconf`, compatibility shims, and C23-targeting code) therefore fail
to preprocess.

### Why `#elifdef` does not accidentally match `#elif`

The existing `#elif` check guards its keyword boundary:

```c
if (strncmp(s + in, "elif", 4) == 0 &&
    !isalnum((unsigned char)s[in + 4]) && s[in + 4] != '_')
```

For `#elifdef`, `s[in + 4]` is `'d'` (alphanumeric) → the condition is
`false`.  For `#elifndef`, `s[in + 4]` is `'n'` → also `false`.  Neither
new directive can be accidentally swallowed by the `#elif` branch, so
placement order is not strictly critical; we place the new branches
*before* `#elif` for consistency with the `#ifdef`-before-`#if` convention.

---

## C standard references

| Directive    | Equivalent to             | Standardised |
|--------------|---------------------------|--------------|
| `#elifdef`   | `#elif defined(NAME)`     | C23 §6.10.1  |
| `#elifndef`  | `#elif !defined(NAME)`    | C23 §6.10.1  |

Both are also accepted as GCC/Clang extensions in C99 mode.

## Task 1 — Add `#elifdef` and `#elifndef` (`src/preprocessor.c`)

### Location

In `preprocess_internal`, the conditional-directive chain runs:

```
#ifdef  (line ~1233)
#ifndef (line ~1257)
#if     (line ~1281)
#elif   (line ~1310)   ← insert new branches immediately before here
#else   (line ~1347)
#endif  (line ~1370)
```

### 1a — `#elifdef NAME`

Insert immediately before the `#elif` block:

```c
/* #elifdef NAME  (C23 §6.10.1 / GCC/Clang extension) */
if (strncmp(s + in, "elifdef", 7) == 0 &&
    !isalnum((unsigned char)s[in + 7]) && s[in + 7] != '_') {
    in += 7;
    if (cond_depth == 0) {
        fprintf(stderr, "error: #elifdef without conditional\n");
        free(out.data); free(spliced); exit(1);
    }
    CondFrame *top = &cond_stack[cond_depth - 1];
    if (top->seen_else) {
        fprintf(stderr, "error: #elifdef after #else\n");
        free(out.data); free(spliced); exit(1);
    }
    int cond_val = 0;
    if (top->parent_emitting && !top->branch_taken) {
        while (s[in] == ' ' || s[in] == '\t') in++;
        size_t name_start = in;
        while (s[in] && (isalnum((unsigned char)s[in]) || s[in] == '_'))
            in++;
        size_t name_len = in - name_start;
        cond_val = (macro_find(macros, s + name_start, name_len) != NULL);
    }
    while (s[in] && s[in] != '\n') in++;
    if (top->parent_emitting) {
        if (!top->branch_taken && cond_val) {
            top->emitting = 1;
            top->branch_taken = 1;
        } else {
            top->emitting = 0;
        }
    }
    emitting = top->emitting;
    continue;
}
```

### 1b — `#elifndef NAME`

Insert immediately after the `#elifdef` block (still before `#elif`):

```c
/* #elifndef NAME  (C23 §6.10.1 / GCC/Clang extension) */
if (strncmp(s + in, "elifndef", 8) == 0 &&
    !isalnum((unsigned char)s[in + 8]) && s[in + 8] != '_') {
    in += 8;
    if (cond_depth == 0) {
        fprintf(stderr, "error: #elifndef without conditional\n");
        free(out.data); free(spliced); exit(1);
    }
    CondFrame *top = &cond_stack[cond_depth - 1];
    if (top->seen_else) {
        fprintf(stderr, "error: #elifndef after #else\n");
        free(out.data); free(spliced); exit(1);
    }
    int cond_val = 0;
    if (top->parent_emitting && !top->branch_taken) {
        while (s[in] == ' ' || s[in] == '\t') in++;
        size_t name_start = in;
        while (s[in] && (isalnum((unsigned char)s[in]) || s[in] == '_'))
            in++;
        size_t name_len = in - name_start;
        cond_val = (macro_find(macros, s + name_start, name_len) == NULL);
    }
    while (s[in] && s[in] != '\n') in++;
    if (top->parent_emitting) {
        if (!top->branch_taken && cond_val) {
            top->emitting = 1;
            top->branch_taken = 1;
        } else {
            top->emitting = 0;
        }
    }
    emitting = top->emitting;
    continue;
}
```

### Design notes

**Inactive-region handling is correct as written.**  Unlike `#include` or
`#define`, branch-transition directives (`#elif`, `#else`, `#endif`, and
now `#elifdef`/`#elifndef`) are checked before the `if (!emitting)` skip
that gates active-only directives.  The two new branches are inserted in
the same pre-emitting-check region, so they correctly update the
`cond_stack` state even when the surrounding block is inactive — which is
required for proper nesting.

**`macro_find` is already visible** at this point in the function (it is
used by `#ifdef` / `#ifndef` earlier in the same `if` chain).  No forward
declarations or include changes are needed.

**The `name_len == 0` edge case** (bare `#elifdef` with no name) will
produce an empty lookup that returns `NULL` — so `#elifdef` with no name
acts as `#elifndef` with no name (both false).  This is consistent with how
`#ifdef` handles a missing name (it finds nothing).  A stricter check is out
of scope for this stage.

## Out of scope — do NOT do these in this stage

- **`#elifdef` in `_Pragma`** — `_Pragma` only handles `"once"` at this
  stage; nested conditional evaluation inside `_Pragma` strings is out of
  scope.
- **`__has_include` / `__has_feature` / `__has_attribute`** — GCC/Clang
  extensions; deferred.
- **`#elifdef` with multiple names or expressions** — only a single
  identifier is accepted (matching C23 semantics); `#elifdef A && B` would
  need to be written as `#elif defined(A) && defined(B)`.
- **`_Noreturn` / `_Static_assert` / `_Alignof`** — unrelated C11 keywords;
  deferred.
- **Floating-point support** — unrelated; deferred.

## Tests

All new tests go in `test/valid/`.

### `test_elifdef_taken__0.c` — `#elifdef` branch is taken

```c
/* The SECOND branch should be taken: BETA is defined, ALPHA is not. */
#define BETA 1
#ifdef ALPHA
#define RESULT 10
#elifdef BETA
#define RESULT 20
#else
#define RESULT 30
#endif
int main(void) { return RESULT - 20; }
```

Expected exit: 0.

### `test_elifdef_else__0.c` — `#elifdef` branch not taken, falls to `#else`

```c
/* Neither ALPHA nor BETA is defined; #else should be taken. */
#ifdef ALPHA
#define RESULT 10
#elifdef BETA
#define RESULT 20
#else
#define RESULT 30
#endif
int main(void) { return RESULT - 30; }
```

Expected exit: 0.

### `test_elifndef_taken__0.c` — `#elifndef` branch is taken

```c
/* ALPHA is defined; BETA is not — #elifndef BETA should be taken. */
#define ALPHA 1
#ifdef ALPHA
#define RESULT 10
#elif defined(ALPHA)
#define RESULT 99
#elifndef BETA
#define RESULT 20
#else
#define RESULT 30
#endif
int main(void) { return RESULT - 10; }
```

Wait — `#ifdef ALPHA` is true, so RESULT is 10 and the remaining branches
are skipped. Revise to exercise `#elifndef`:

```c
/* ALPHA is not defined; BETA is not defined — #elifndef BETA is taken. */
#ifdef ALPHA
#define RESULT 10
#elifndef BETA
#define RESULT 20
#else
#define RESULT 30
#endif
int main(void) { return RESULT - 20; }
```

Expected exit: 0.

### `test_elifndef_false__0.c` — `#elifndef` branch not taken because NAME is defined

```c
/* ALPHA not defined; BETA is defined — #elifndef BETA is false, #else taken. */
```

### `test_elifdef_nested__0.c` — `#elifdef` inside a nested conditional

