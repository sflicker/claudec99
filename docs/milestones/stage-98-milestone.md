# Milestone Summary

## Stage 98: Compound Literals — Complete

Stage 98 ships C99 compound literals — the `(type-name){ initializer-list }` syntax for unnamed temporary objects allocated on the stack — enabling code like `(int){42}`, `(struct Point){3, 4}`, and `(int[]){1,2,3}` with or without explicit sizes, and supporting postfix operators and address-of on the result.

- **AST**: Added `AST_COMPOUND_LITERAL` node type with `decl_type` (base kind), `full_type` (struct/union/array/pointer Type*), `byte_length` (rbp-relative stack offset, assigned during codegen pre-scan), and `children[0]` (AST_INITIALIZER_LIST or bare expression).
- **Parser**: Implemented `build_compound_literal()` helper to construct compound literals from a parsed type and brace-initializer; added `infer_compound_literal_array_length()` to walk designator and positional initializers for omitted-size arrays; extended `parse_cast` to detect `(type-name){` after consuming type; extended `parse_postfix` with save/restore lexer state before `(` to distinguish `(type-name){` from ordinary casts. New `parse_postfix_suffixes()` helper applies trailing operators (`[]`, `.`, `->`, `()`, `++`, `--`) to compound-literal results and cast expressions. Updated `parse_type_name` to accept `[]` (omitted first dimension) for compound literals while rejecting it for `sizeof` and cast expressions. Updated `parse_unary` to accept compound literals as lvalues for address-of (`&`). Global initializer path rejects file-scope compound literals.
- **Codegen**: Two-phase pre-scan: `compute_compound_literal_bytes()` estimates stack space; `scan_expr_compound_literals()` assigns each compound literal a concrete rbp-relative offset. Four emission sites: struct/union → `emit_local_struct_init`; array → local array initializer path; scalar → direct store; address-of struct/union/array → address already in rax, address-of scalar → `lea rax, [rbp - offset]`. Updated `emit_struct_addr` to return correct base type for compound literals. Result types set to pointer-to-base for address-of expressions.
- **Tests**: 20 new tests — 12 valid (struct arg/assign, explicit/omitted array size, array designators, postfix member/subscript, address-of, scalar, zero-fill, dot-product, char array), 4 invalid (file-scope, void type, too-many initializers, oob designator), 3 print_ast (struct/array/scalar), 1 integration multifile.
- **All 1521 tests pass** (855 valid, 246 invalid, 86 integration, 48 print_ast, 100 print_tokens, 21 print_asm). Self-host C0→C1→C2 cycle passes cleanly; no bootstrap bugs found.
- **Version**: Bumped VERSION_STAGE to `"00980000"`.
- **Notes**: Compound literals at file scope are not yet supported (error issued at parse time). Save/restore of lexer state before `(` in `parse_postfix` distinguishes postfix compound literals from calls. Omitted array size inferred from initializer designators; explicit array size bounds-checked against initializer. Scalar compound literals in braces treated as single-value initializers (e.g., `(int){42}` == `42`).
