prefix/postfix `++`/`--` **on any modifiable lvalue**; unary `&` (address-of) **on any addressable lvalue** including `&s.member`, `&p->member`, `&arr[i].member`, and **`&(T){ … }` compound plus **forward-declared enum tags** (`typedef enum Status Status;` before the body — stage 99); structs (definition, natural-alignment layout, `.`/`->` access with **general postfix chaining** like structs, typedef aliases, incomplete forward declarations); **named unions** (layout = max member size, member access, nested types, whole-union assignment, globals); **anonymous struct/union type Lexer/Parser/CodeGen/AST pipeline with full per-file heap teardown (`parser_free`, `codegen_free`, `lexer_free`, `ast_free`, `reset_error_state`); with `--max-errors=0` a compile error in one file Diagnostics carry `<file>:<line>:<col>:` position prefixes on parser errors, support `--max-errors=N` multi-error collection with setjmp/longjmp recovery, expose a two-line `--version` build identifier with a `BuiltBy:` attribution field, and honor `-Werror`. A **multi-mode build workflow** (`build.sh`: `--mode=normal`/`--mode=bootstrap`/`--mode=fallback`/`--mode=self-host`, with bootstrap timeout guards) drives builds; the compiler's own source is strict ISO C99, and as of stage 92 the compiler is **fully self-hosting** — C0 (gcc-built) compiles itself to C1 and C1 compiles itself to C2, with all 1621 tests passing at each step (see `docs/self-compilation-report.md`). Stages 94 and 95-01 through 95-12 added no language features; they harden the self-host cycle and complete an internal **dynamic-storage refactor** — new `Vec` and `StrBuf` containers replace every fixed-capacity parser/codegen table, and all token, AST, parser, and codegen name/tag/label text moves from `char[MAX_NAME_LEN]` buffers into `const char *` pointers backed by a lexer-owned string pool, removing the old per-table caps and the 255-byte string-literal limit. Stage 96 adds multi-file compilation and per-file teardown with no language changes. Stage 97 adds **designated initializers** (`.member = value` and `[index] = value`). Stage 98 adds **compound literals** (`(T){ … }` for struct, array, and scalar types). Stage 99 completes **`typedef enum`** — full integer constant expression enumerator values and forward-declared enum tags. Stages 100–104 progressively enhance static local support and the constant-expression evaluator. Stage 105 adds **C99 preprocessor completion** (`#pragma`, `_Pragma`, `#line`, null directive, `__func__`). Stage 106 adds **C99 header completion** (`restrict` qualifier as parse-and-ignore; expanded stubs for `<ctype.h>`, `<string.h>`, `<stdlib.h>`, `<stdio.h>`) and fixes a pre-existing long-long codegen bug. Stage 107 adds **`inline` keyword parsing** (parse-and-ignore), a **`<assert.h>` stub** with an NDEBUG-aware assert macro, and **functional `va_copy` codegen** (three 8-byte moves for the 24-byte `va_list`), along with a preprocessor fix enabling `__FILE__`/`__LINE__` expansion inside function-like macro bodies. Stage 108 adds **`#elifdef`/`#elifndef`** branch-transition directives (C23 §6.10.1 / GCC/Clang extension), eliminating a gap that caused real-world headers using these directives to fail with "unsupported preprocessor directive". All 1621 tests pass at C0, C1, and C2.

What's still out of scope: floating-point types (`float`, `double`) and all floating-point arithmetic/literals/conversions; `va_arg` for floating-point and struct-by-value types; multi-character and wide-character literals; compound literals at file scope; bit-fields, flexible array members; `volatile` code-generation semantics (currently parsed and tracked only); chained designators (`.a.b`, `.arr[0]`) and designated union init for non-first members; functions returning function pointers; pointer-to-array declarators (`(*p)[10]`); old-style (K&R) function definitions; `__attribute__` specifiers; GNU variadic macro extensions (`__VA_OPT__`, named variadic args, comma deletion); object-file (`.o`) emission and separate linking; header-only precompilation.
