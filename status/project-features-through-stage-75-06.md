What works today: a full preprocessing pipeline (line splicing, comment removal, `#include "file.h"` and `#include <file.h>`, object-like and function-like `#define` with parameter substitution and recursive expansion, variadic macros with `__VA_ARGS__`, macro stringification `#param`, token pasting `##`, `#undef`, conditional compilation `#ifdef`/`#ifndef`/`#if`/`#elif`/`#else`/`#endif` with full recursive-descent expression evaluation including `defined()`, macro expansion, arithmetic, bitwise, shift, logical, and comparison operators, `#error`, predefined macros `__FILE__`/`__LINE__`/`__DATE__`/`__TIME__`/`__STDC__`/`__STDC_VERSION__`/`__STDC_HOSTED__`/`__CLAUDEC99__`, ABI-reflecting macros `__x86_64__`/`__linux__`/`__unix__`/`__LP64__`/`_LP64`/`__CHAR_BIT__`/the full `__SIZEOF_*__` family, command-line `-D`/`-I`/`--sysroot` flags); an expanded set of stub system headers in `test/include/` covering `stdio.h` (opaque `FILE`, `EOF`, `fopen`, `fclose`, `fgetc`, `fgets`, `fprintf`, `snprintf`, and the `vfprintf`/`vprintf`/`vsnprintf` variadic-forwarding family), `stddef.h`, `stdlib.h` (`malloc`, `free`, `exit`, `realloc`), `string.h` (`strlen`, `strcmp`, `strcpy`, `memcpy`, `memset`, `memcmp`, `strchr`), `limits.h` (with `LLONG_MIN`/`LLONG_MAX`/`ULLONG_MAX`), `stdint.h` (exact-width/pointer-size/fast/least integer typedefs), `stdbool.h` (`bool`, `true`, `false`), `ctype.h` (character classification), `errno.h` (`errno`, `ERANGE`, `EINVAL`, …), `time.h` (`time_t`, time functions), `setjmp.h` (non-local jumps), and `stdarg.h` (`va_list` plus the `va_start`/`va_end`/`va_copy`/`va_arg` macros); statements (if/else, while/do-while/for, switch/case/default, break/continue/goto, blocks with shadowing, expression statements); the full signed integer hierarchy (`char`/`short`/`int`/`long`) plus all four unsigned variants plus `long long` and `unsigned long long`, the `signed` keyword forms and trailing-`int` forms; `_Bool` with value-normalization and integer promotion; unsigned-aware arithmetic, division, shifts, and comparisons; integer promotions and Usual Arithmetic Conversions for mixed signed/unsigned operands including LP64-specific rules; `U`/`L`/`UL`/`LL`/`ULL` literal suffixes with overflow-aware typing; void type and `void*` generic pointer; minimal `const` qualifier on base-type scalars **plus pointer-level `const` enforcement** (writes through `const*` rejected, const-pointer reassignment rejected, const-discard conversions warned or errored under `-Werror`); casts; multi-function programs with forward declarations; **variadic function definitions** with `<stdarg.h>` support — `va_list`, `va_start`, `va_end`, and `va_arg` for GP-class types (`int`/`unsigned`/`long`/`unsigned long`/`long long`/`unsigned long long`/pointer) via a SysV AMD64 register save area with both register and overflow-stack paths; **function calls with an arbitrary number of arguments** (the 6-argument limit is gone — full System V AMD64 stack-passing on both caller and callee sides, with 16-byte alignment); pointers (declarations, `&`/`*` as rvalue and lvalue, assign-through-pointer, pointer parameters and return types, full pointer arithmetic including difference yielding `long`, `NULL`, and null-constant `0` accepted as a pointer argument); arrays (declarations, indexing, multi-level subscript, decay-to-pointer, file-scope brace/string initializers with size inference, array-to-pointer parameter decay); string and character literals (tokenization, static-data emission, decoded simple escapes); all bitwise operators, remainder, and all eleven compound assignment operators; `sizeof`; conditional (ternary) and comma operators; comma-separated init-declarator lists; file-scope global variable declarations; storage-class specifiers `extern` and `static` at file scope **and block-scope `static` local variables** (persist across calls; `.bss`/`.data` with RIP-relative addressing); parenthesized and abstract declarators; function pointer declarations/assignment/indirect calls; `typedef` aliases for scalar/pointer/array/function-pointer and complete struct/union types; `enum` declarations with compile-time constant folding; structs (definition, natural-alignment layout, `.`/`->` access, brace initializers, whole-struct assignment/copy, pointer-to-struct mutation, nested structs, arrays of structs, typedef aliases, incomplete forward declarations); **named unions** (layout = max member size, member access, nested types, whole-union assignment, globals); and **anonymous struct/union type declarations** (each definition is a unique type identified by pointer). Diagnostics now carry `<file>:<line>:<col>:` position prefixes on parser errors, support `--max-errors=N` multi-error collection with setjmp/longjmp recovery, expose a `--version` build identifier, and honor `-Werror`.

What's still out of scope: floating-point types (`float`, `double`) and all floating-point arithmetic/literals/conversions; `for`-loop initializer declarations (`for (int i = 0; …)`); `va_arg` for floating-point and struct-by-value types, and `va_copy` codegen (still a no-op stub); octal and hex character/string escapes (`'\123'`, `'\x41'`); multi-character and wide-character literals; struct by-value parameters and return values (hidden-pointer ABI); bit-fields, flexible array members, compound literals; `volatile` and `restrict` qualifiers; `typedef enum`; enum constant expressions beyond integer/character literals; enum constants and other non-literal constant expressions as `switch`/`case` labels; subscripting or postfix-`++`/`--` on member-access and other non-identifier operands, and compound assignment to non-identifier lvalues (parser limitations surfaced by self-compilation); functions returning function pointers; `__attribute__` specifiers; `#pragma` (including `#pragma once`); `#elifdef`/`#elifndef`; GNU variadic macro extensions (`__VA_OPT__`, named variadic args, comma deletion); object-file emission, linking, and multi-file `.o` compilation.
